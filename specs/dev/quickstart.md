# Quickstart: Public Gallery & Admin Dashboard

**Feature**: Public Gallery & Admin Dashboard
**Date**: 2026-03-13

## 前置條件

1. Supabase 專案已建立且包含 `gallery_items` 資料表
2. Supabase Storage bucket 已建立並設定公開存取
3. `.env` 已設定 `SUPABASE_URL` 與 `SUPABASE_KEY`
4. Google OAuth provider 已在 Supabase Auth 設定中啟用

## 建置驗證

```bash
# Step 1: Nuxt 準備（產生型別）
npx nuxi prepare

# Step 2: 建置專案
npm run build
```

兩個命令皆應無錯誤完成。

## 手動驗證步驟

### V-001: 首頁 Hero Section

1. 啟動開發伺服器：`npm run dev`
2. 開啟瀏覽器至 `http://localhost:3000`
3. **預期**：頁面頂部顯示 Hero 區塊，包含網站主題描述與圖片跑馬燈
4. **預期**：桌面視窗寬度（≥768px）顯示三欄垂直跑馬燈
5. **預期**：手機視窗寬度（<768px）顯示水平跑馬燈
6. **對應 FR**：FR-020

### V-002: 公開畫廊瀏覽

1. 在 Supabase 插入 15+ 筆 `isActive: true` 的測試資料
2. 重新載入首頁
3. **預期**：畫廊網格僅顯示 active 項目，依建立時間降冪排列
4. **預期**：向下捲動時自動載入更多項目（無限捲動）
5. **預期**：載入中顯示 skeleton placeholder
6. **對應 FR**：FR-001, FR-002, FR-003, FR-004

### V-003: 畫廊細節彈窗

1. 點擊任一畫廊卡片
2. **預期**：Modal 彈窗開啟，顯示完整圖片、標題、prompt、badges、建立日期
3. 點擊「Copy Prompt」按鈕
4. **預期**：剪貼簿中包含 prompt 文字，顯示成功提示
5. **對應 FR**：FR-005, FR-006

### V-004: 空畫廊狀態

1. 將所有項目設為 `isActive: false`
2. 重新載入首頁
3. **預期**：顯示友善的空狀態訊息
4. **對應 FR**：FR-007

### V-005: 登入功能

1. 瀏覽至 `/login`
2. 點擊 Google 登入按鈕
3. **預期**：導向 Google OAuth 流程
4. 完成 OAuth 後
5. **預期**：自動導回應用程式
6. **對應 FR**：FR-016

### V-006: 存取控制

1. 登出（或開啟無痕視窗）
2. 直接瀏覽至 `/admin`
3. **預期**：自動導向 `/login`
4. **對應 FR**：FR-016, FR-017, SC-008

### V-007: 管理列表

1. 以 admin 身份登入
2. 瀏覽至 `/admin`
3. **預期**：UTable 顯示所有項目（包含 inactive），每列有縮圖、標題、prompt、日期、狀態、badges、操作按鈕
4. **對應 FR**：FR-008

### V-008: 新增項目

1. 在管理頁面點擊「新增」按鈕
2. **預期**：USlideover 開啟，列表仍在背景可見
3. 上傳 .webp 圖片（≤2MB）、填入標題、prompt、新增至少一個 badge、設定 isActive
4. 提交表單
5. **預期**：Toast 顯示成功通知，列表自動更新
6. **對應 FR**：FR-009, FR-012, FR-013, FR-014

### V-009: 編輯項目

1. 在列表中點擊某項目的「編輯」按鈕
2. **預期**：USlideover 開啟，表單預填現有資料
3. 修改標題並提交
4. **預期**：Toast 顯示成功通知，列表反映變更
5. 不上傳新圖片時提交
6. **預期**：原有圖片保留不變
7. **對應 FR**：FR-010, FR-012

### V-010: 刪除項目

1. 在列表中點擊某項目的「刪除」按鈕
2. **預期**：確認對話框出現
3. 點擊取消
4. **預期**：項目未被刪除
5. 再次點擊刪除 → 確認
6. **預期**：Toast 顯示成功，項目從列表消失
7. **對應 FR**：FR-011, FR-012, SC-006

### V-011: 表單驗證

1. 在新增表單中不上傳圖片，直接提交
2. **預期**：顯示圖片必填錯誤
3. 不填標題、prompt，或不加 badge，分別提交
4. **預期**：各欄位顯示相應驗證錯誤
5. **對應 FR**：FR-015

### V-012: 圖片上傳失敗處理

1. 上傳一個非 .webp 格式的檔案
2. **預期**：驗證阻擋，提示格式錯誤
3. 模擬 URL 驗證失敗的情境
4. **預期**：提交被阻斷，Toast 顯示錯誤，無資料寫入
5. **對應 FR**：FR-015, SC-010

### V-013: 響應式驗證

1. 使用瀏覽器 DevTools 切換至手機尺寸（< 768px）
2. 瀏覽首頁、管理頁面
3. **預期**：所有頁面在各尺寸下正常顯示與操作
4. **對應 FR**：FR-018, SC-009
